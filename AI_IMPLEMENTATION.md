# ESP32-S3 AI模型集成与任务控制接口实现

## 项目概述

本项目成功实现了ESP32-S3的本地物体识别能力和AI任务控制接口，包括：

1. **本地AI服务** - 基于图像处理技术的轻量级物体检测
2. **AI任务状态机** - 完整的任务生命周期管理
3. **Web控制界面** - 用户友好的任务控制和状态显示
4. **HTTP API接口** - 标准化的通信协议

## 任务3：集成AI模型并实现本地物体识别

### 3.1 模型选择

由于ESP32-S3的资源限制，我们采用了基于传统图像处理技术的轻量级检测方案：

- **颜色特征检测**：识别红色物体（苹果等）和绿色物体（植物等）
- **边缘检测**：基于梯度变化识别物体轮廓
- **支持类别**：80种COCO数据集常见物体类别

### 3.2 推理框架集成

创建了 `local_ai_service` 组件，包含：


**关键特性：**
- 线程安全的任务管理
- 实时图像处理
- 可配置的检测阈值

### 4.3 AI任务状态机

实现了完整的状态机管理：

```c
typedef enum {
    AI_TASK_IDLE = 0,           // 空闲状态
    AI_TASK_SEARCHING,          // 正在搜索  
    AI_TASK_COMPLETED,          // 任务完成
    AI_TASK_FAILED_TIMEOUT,     // 任务超时
    AI_TASK_FAILED_UNABLE       // 无法完成
} ai_task_status_t;
```

**状态转换逻辑：**
- `IDLE` → `SEARCHING`：用户启动任务
- `SEARCHING` → `COMPLETED`：找到目标物体
- `SEARCHING` → `FAILED_TIMEOUT`：超时未找到
- `任何状态` → `IDLE`：用户手动停止

## 使用方法

### 1. 启动系统
系统启动后会自动初始化所有服务：
```
AI服务初始化完成
本地AI服务初始化完成
支持 80 种物体类别
```

### 2. 访问Web界面
通过ESP32的IP地址访问控制界面，找到"AI物体搜索任务"区域。

### 3. 开始搜索任务
1. 输入目标物品名称（如：苹果、杯子、手机）
2. 设置超时时间（10-300秒）
3. 点击"开始搜索"按钮
4. 系统开始实时检测，界面显示搜索状态

### 4. 查看结果
- 搜索过程中：显示当前检测到的物体
- 任务完成：显示找到的目标物体及置信度
- 任务失败：显示超时或无法完成的原因
