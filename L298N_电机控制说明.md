# ESP32-S3 摄像头 + L298N电机控制系统

## 项目概述

这个项目集成了ESP32-S3摄像头AI分析和L298N电机驱动功能，实现了智能视觉识别和运动控制的结合。

## L298N电机控制功能

### 硬件连接

L298N电机驱动模块与ESP32-S3的连接方式：

| L298N引脚 | ESP32-S3引脚 | 说明 |
|-----------|-------------|------|
| IN1       | GPIO 3      | 左电机正转控制 |
| IN2       | GPIO 1      | 左电机反转控制 |
| IN3       | GPIO 2      | 右电机正转控制 |
| IN4       | GPIO 42     | 右电机反转控制 |
| VCC       | 5V          | 电源正极 |
| GND       | GND         | 电源负极 |

### 电机控制API

#### 基本运动控制函数

```c
// 电机前进
motor_driver_init();          // 初始化电机驱动
motor_forward(60);           // 前进，速度60%
motor_backward(60);          // 后退，速度60%
motor_left(60);              // 左转，速度60%
motor_right(60);             // 右转，速度60%
motor_stop_all();            // 停止所有电机
```

#### 电机测试功能

系统启动时会自动执行电机测试序列：
1. 前进 2秒
2. 停止 1秒
3. 后退 2秒
4. 停止 1秒
5. 左转 1.5秒
6. 停止 1秒
7. 右转 1.5秒
8. 最终停止

### 电机运动逻辑

- **前进**: 左电机和右电机同时正转
- **后退**: 左电机和右电机同时反转
- **左转**: 左电机反转，右电机正转
- **右转**: 左电机正转，右电机反转
- **停止**: 所有电机停止

### 系统启动流程

1. 初始化NVS和SPIFFS存储
2. 启动WiFi AP+STA双模式
3. 初始化SNTP时间同步
4. 初始化摄像头
5. **初始化L298N电机驱动**
6. 启动Web服务器
7. **启动电机测试任务**
8. 启动摄像头AI分析任务

### 系统特性

- **双模式网络**: 同时支持连接现有WiFi和创建热点
- **智能视觉**: AI分析摄像头图像内容
- **电机控制**: L298N驱动双电机差动转向
- **Web界面**: 可通过浏览器查看图像历史
- **实时日志**: 串口输出详细运行状态

### 注意事项

⚠️ **重要提醒**：
- 确保L298N电源供应充足（5V，建议2A以上）
- 检查所有GPIO连接是否正确
- 电机测试时确保设备安全放置
- 避免长时间连续运行导致过热

### 故障排除

1. **电机不转动**：
   - 检查L298N电源连接
   - 确认GPIO引脚连接正确
   - 查看串口日志确认初始化成功

2. **电机转向错误**：
   - 交换对应电机的IN1和IN2引脚
   - 或修改代码中的电机控制逻辑

3. **系统重启**：
   - 可能是电源不足
   - 检查是否有短路现象

现在系统具备完整的摄像头AI分析 + L298N电机控制功能，可以实现智能移动机器人的基础功能。通过AI识别环境内容，结合电机控制实现自主导航和响应。
